# Changelog

## 1.6.0

- Оновлено дизайн усієї адмінки: інформативні заголовки сторінок, єдині UI-токени, компактні картки, чистіші таблиці, поля та responsive-стани.
- Маршрути інтеграцій у редакторі форми тепер початково згорнуті й показують короткий опис налаштувань.
- Додано версійні маршрути Telegram, Google Sheets і CRM для кожної форми з успадкуванням глобальних підключень.
- Кожна нова доставка зберігає незмінний snapshot маршруту; старі jobs без snapshot працюють через сумісний global resolver.
- Додано мовні Telegram-шаблони, режими plain text, HTML і MarkdownV2, topic ID, inline-кнопки та безпечний preview.
- Розширено Google Sheets: завантаження вкладок, створення аркуша, візуальне зіставлення колонок, RAW-запис заголовків, append та update за точним збігом.
- Додано тестові заявки через штатну чергу. Вони мають ознаку `is_test` і не враховуються в dashboard.
- Додано contextual connector API та `Delivery_Request`, а старий контракт конекторів залишено сумісним.
- Розділено конектори, Google API client, Telegram renderer, repository-домени та assets редактора інтеграцій.
- Додано dependency-free PHP-тести для Telegram renderer, mapping, A1 ranges і route sanitization.

## 1.5.1

- Додано явні capability checks для кожного адміністративного екрана та POST-only перевірку всіх операцій, що змінюють дані.
- Дані заявки повторно санітизуються й обмежуються після сторонніх PHP-фільтрів перед записом у базу та передачею інтеграціям.
- Серверний API створення заявок генерує унікальний request ID замість повторного порожнього значення.
- Запити з токенами до Telegram, Google та CRM не переходять за HTTP-редиректами й мають обмежений розмір відповіді.
- Системне попередження про старі плагіни відображається лише адміністраторам.


## 1.5.0

- Google Service Account JSON тепер завантажується, замінюється та видаляється безпосередньо в адмінці.
- JSON перевіряється на сервері, шифрується AES-256-GCM і не зберігається як публічний файл.
- Повне посилання Google-таблиці автоматично перетворюється на Spreadsheet ID.
- Новий трикроковий майстер показує email Service Account для надання доступу Editor.
- Старий `LEADFORMS_GO_GOOGLE_CREDENTIALS_PATH` залишається необов’язковою сумісністю.

## 1.4.4

- Блок Google Service Account перетворено на покроковий майстер підключення Google Sheets.
- Додано статус JSON-файлу, прямі посилання Google Cloud, рекомендований безпечний шлях і кнопку копіювання конфігурації.
- В адмінці пояснено Spreadsheet ID, назву аркуша, порядок полів, доступ Editor і фінальну перевірку.

## 1.4.3

- Історія та деталі заявки показують локалізовані підписи полів і зрозумілі значення checkbox.
- Стандартні повідомлення форми автоматично заповнюються для кожної мови й залишаються доступними для редагування.
- Нові поля з плиток одразу отримують українські та англійські підписи й placeholder.

## 1.4.2

- Telegram використовує локалізовані підписи полів візуальної форми замість технічних ключів.
- Значення checkbox у Telegram відображається як «Так» або «Ні» відповідно до мови заявки.
- Required-checkbox перевіряється сервером як логічне значення, а не як довільний непорожній рядок.
- Нова доставка обробляється серверним fallback одразу після відповіді форми, якщо WP-Cron не запустився.
- Збережено блокування черги та атомарне захоплення доставки для захисту від дублювання.

## 1.4.1

- Усунено виконання формул із публічних полів у Google Sheets через режим `RAW`.
- Додано активність форми, per-form nonce, підписаний контекст, honeypot і мінімальний час заповнення.
- Rate limiting перенесено в атомарну таблицю з окремими IP та глобальними лімітами.
- Додано request idempotency: повтор одного запиту не створює нову заявку або доставку.
- Згенеровані та кодові форми примусово використовують безпечний POST fallback без зовнішнього action.
- Telegram і CRM токени та кешований Google access token шифруються у базі; підтримано constants у `wp-config.php`.
- Перервана доставка більше не повторюється автоматично з невідомим результатом.
- Додано retention заявок, строк зберігання UTM, маскування списку та WordPress Privacy exporter/eraser.
- Видалення форми переведено з GET на POST, а release build застосовує `.distignore` рекурсивно.
- Кодові форми приймають лише поля, які реально описані в HTML форми; сторонні ключі відхиляються.
- Перевірка `Origin` враховує протокол і порт, а не лише домен.
- Privacy eraser обробляє великі набори заявок пакетами без завантаження всієї історії в пам’ять.
- Помилка шифрування нового секрету більше не стирає попереднє робоче значення.

## 1.4.0

- Додано одну структуру форми з окремими перекладами полів, кнопки та повідомлень за locale.
- Додано мовні вкладки, індикатор заповненості, копіювання з основної мови та попередній перегляд.
- Shortcode підтримує `locale`, а мова також визначається через Polylang або WordPress locale.
- Заявки зберігають locale, а інтеграції отримують стабільні технічні ключі незалежно від мови.
- Додано ідемпотентну міграцію поточних візуальних форм і fallback на основну мову.


## 1.3.4

- Копіювання shortcode більше не замінює текст кнопки та не змінює ширину таблиці.

## 1.3.3

- Виправлено копіювання shortcode у браузерах без доступу до Clipboard API.
- Додано видиме підтвердження успішного або невдалого копіювання.
- Після збереження форми показується повідомлення про успіх.
- У редакторі форми додано кнопку повернення до списку форм.

## 1.3.2

- Додано серверний fallback для прострочених доставок, якщо loopback WP-Cron не виконується.
- Fallback використовує наявний lock і атомарний `claim_delivery()`, тому одночасний запуск із cron не дублює доставку.
- Прострочені cron-події тепер коректно визначаються та переплановуються.
- Dashboard показує кількість прострочених доставок, джерело останнього запуску й ознаку простроченої cron-події.
- Нові успішні доставки отримують статус `sent`; наявний статус `success` залишається сумісним з історією та статистикою.
