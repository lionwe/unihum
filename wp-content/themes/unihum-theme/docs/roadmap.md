# UNIHUM Theme Roadmap

Дата аудиту: 16 липня 2026

## Поточний стан

Тема має робочий технічний каркас: PHP-шаблони, SCF JSON, Webpack-збірку, базові SCSS-парціали, підключення Google Fonts і початкові кольорові токени.

Водночас це ще не готова стартова тема для лендингу. На цей момент у фронтенді реалізовано лише порожній шаблон головної сторінки та заглушки для hero, header і footer.

## Готово

- Класична WordPress-тема `unihum-theme` із метаданими в `style.css`.
- Підключення production CSS і JavaScript з cache busting через `filemtime()`.
- Webpack 5 та команда `npm run build`.
- Базова SCSS-архітектура: `base`, `layout`, `components`, `utils`, `vendors`.
- Google Fonts: Inter і Cormorant Garamond.
- Кольорові токени `primary`, `secondary`, `neutral`, `white`, `black`, `red`.
- Підтримка title tag, featured image, custom logo та меню header/footer.
- SCF JSON load/save у `scf-fields/`.
- Порожня група SCF для Home і наявна група Site Options.
- Базовий шаблон кнопки із санітизацією URL, тексту, класів і target.

## Обов'язково до першої робочої версії

1. **Визначити MVP-структуру головної сторінки.**
   - Рекомендовані секції: Hero, центральне питання, чому зараз, три принципи, позиція щодо AI, CTA/contact, footer.
   - Не переносити у MVP документи, новини, roadmap 2026–2045 або сторінку «спільноти», якщо для них немає затвердженого контенту.

2. **Реалізувати layout.**
   - Замість текстових заглушок створити доступні `header.php` і `footer.php`.
   - Вивести зареєстровані меню, логотип, мовний перемикач лише після визначення механіки мультимовності.
   - Створити секційні PHP-шаблони та відповідні SCSS-парціали за структурою теми.

3. **Створити SCF-поля для MVP.**
   - Hero: eyebrow, title, lead, CTA label/link, фонове зображення або графіка.
   - Контентні блоки: заголовок, текст, repeaters для принципів/карток.
   - Footer: контакт, посилання на соцмережі, copyright.
   - Для зображень використовувати ID/array, а не URL-рядки.

4. **Визначити контактний сценарій.**
   - Підтвердити: форма Contact Form 7, e-mail або інший канал.
   - У `functions.php` зареєстрований callback `send_message_to_telegram`, але функції в темі немає. До підключення форми його треба реалізувати або прибрати цей hook.

5. **Додати та перевірити контент.**
   - Підготувати затверджені тексти першої мови сайту.
   - Додати логотип у SVG та ліцензовані медіа.
   - Узгодити мови MVP і спосіб мультимовності до створення навігації.

6. **QA перед запуском.**
   - Перевірити головну на desktop, tablet і mobile.
   - Перевірити клавіатурну навігацію, focus states, alt-тексти, контраст і `prefers-reduced-motion`.
   - Перевірити форму: nonce, spam protection, статуси success/error і фактичну доставку.
   - Запустити `npm run build`, `git diff --check` та PHP syntax check через PHP binary Local.

## Порядок реалізації

| Етап | Результат | Залежність |
| --- | --- | --- |
| 1. Контент і IA | Затверджені секції, текст і CTA | Рішення клієнта щодо мов, контакту, автора |
| 2. Базовий layout | Header, footer, navigation, container rules | Логотип і меню |
| 3. Hero | Перший екран із SCF-полями | Затверджений content і hero visual |
| 4. Змістові секції | Question, principles, AI position | Тексти й іконки/графіка |
| 5. Contact | Форма або інший контактний сценарій | Обраний канал і дані отримувача |
| 6. QA | Responsive, accessibility, форма, build | Завершена верстка |

## Не робити на MVP

- Особисті кабінети, AI-чат, базу документів і складну платформу.
- Велику дорожню карту як навігаційний розділ.
- Будь-які непідтверджені лічильники учасників, партнерів або досягнень.
- Нові залежності без потреби конкретної функції.
