# UNIHUM Theme Style Guide

Дата: 16 липня 2026

## Візуальний напрям

**Humanist Editorial Futurism.** Сайт має виглядати як відкрита інтелектуальна ініціатива про майбутнє людства: редакційно, світло, спокійно та сучасно. Це не AI-стартап, не езотеричний проєкт і не великий інформаційний портал.

## Кольори

Токени визначені у `assets/css/base/_variables.scss`.

| Токен | Значення | Використання |
| --- | --- | --- |
| `--color-primary` | `#F3C84B` | Головна CTA, акценти, номерування. |
| `--color-primary-dark` | `#D6A72D` | Hover/focus primary CTA. |
| `--color-primary-light` | `#FFF3C9` | Теплі виділені площини. |
| `--color-secondary` | `#A9D9F5` | Hero і великі блакитні площини. |
| `--color-secondary-light` | `#EAF7FF` | Підкладки, контролі, mobile menu. |
| `--color-secondary-muted` | `#C8D7DE` | Межі та роздільники. |
| `--color-neutral-950` | `#17263A` | Основний текст і контурна графіка. |
| `--color-neutral-600` | `#526274` | Другорядний текст. |
| `--color-neutral-50` | `#FFFCF4` | Основний світлий фон. |
| `--color-white` | `#FFFFFF` | Контрастні поверхні та текст на темному медіа. |
| `--color-red-600` | `#EE0606` | Лише помилки форми та критичні повідомлення. |

Правила:

- Блакитний веде екран від верхньої частини; жовтий підсилює цільову дію або завершує секцію.
- Жовтий не застосовується для довгого тексту або великих темних блоків.
- Основний фон — `neutral-50`, не чистий білий.
- Не додавати нові hex-значення в компоненти: спочатку додати токен у `_variables.scss`.

## Типографіка

Шрифти підключені в `functions.php` через Google Fonts.

| Роль | Гарнітура | Вага |
| --- | --- | --- |
| Headings / logo | Cormorant Garamond | 600, 700 |
| Body / UI / navigation | Inter | 400, 500, 600, 700 |

- `$font-primary`: Inter.
- `$font-secondary`: Cormorant Garamond.
- Для тексту інтерфейсу застосовувати `$font-primary`.
- Для H1–H3, великих цитат і редакційних тез застосовувати `$font-secondary`.
- Не використовувати Cormorant Garamond для довгих абзаців, форм або меню.

## Layout

- Внутрішній контент секції обов'язково розташовується у `.container`.
- Поточна максимальна ширина контейнера — 90rem; змінювати її тільки як глобальний токен.
- Базовий вертикальний gap секцій — `--_section-gap`.
- На desktop використовувати 12-колонкову логіку, на tablet — 6, на mobile — 4.
- Довгі текстові колонки не повинні перевищувати приблизно 680px.
- Між змістовими секціями переважно працювати простором і тонкою лінією, а не тінями або рамками карток.

## Компоненти

### Header

- Логотип ліворуч, навігація праворуч.
- У MVP дозволені лише наявні й робочі пункти меню.
- На mobile — доступне toggle-меню з керованим focus state.

### Buttons

- Primary: `--color-primary` із темним текстом.
- Hover/focus: `--color-primary-dark` із контрастним станом focus.
- Secondary: прозора або світла кнопка з `--color-neutral-950` і тонкою межею.
- Всі кнопки мають видимий focus state і зрозумілий текст дії.

### Cards

- Використовувати тільки для повторюваних коротких одиниць контенту.
- Без glassmorphism, великих shadow, неонових градієнтів і SaaS-подібних «feature cards».
- Базова композиція: номер або малий label, заголовок, короткий опис, тонка верхня межа.

### Forms

- Світле поле на блакитній підкладці, темний текст, межа `--color-neutral-950`.
- Помилки — лише `--color-red-600`.
- Не покладатися на колір як на єдиний індикатор помилки.

## Media та графіка

Дозволено:

- спокійні документальні фото людей, ландшафтів, архітектури знань;
- горизонт, світло, топографічні й орбітальні тонкі лінії;
- сонячний знак тільки як акцент або бренд-елемент.

Заборонено:

- роботизовані руки, мозок із дротами, HUD, неон і 3D chrome;
- постійне «зоряне поле» та генеричне зображення Землі з мережевими точками;
- релігійні або езотеричні образи, що можуть змінити позиціонування ініціативи.

## Motion і доступність

- UI transition: 180–320 ms; появи блоків: до 700 ms.
- Допустимі fade, невелике зміщення вгору та прокреслювання тонкої лінії.
- Усі анімації повинні поважати `prefers-reduced-motion`.
- Для всіх інтерактивних елементів потрібні keyboard focus states.
- Динамічні WordPress-зображення рендерити через `wp_get_attachment_image()` з осмисленим alt.
